You are working on an ESP-IDF + LVGL application for the Elecrow CrowPanel Advance 7.0" ESP32-S3 display, PCB revision V1.3 (SKU DIS02170A).

The current project is CryptoTracker for Elecrow CrowPanel Advance 7.0. It already includes:
- Dashboard
- CoinWatch
- Coin Detail
- News
- Add Coin
- Settings
- OTA update flow
- Alerts, WiFi manager, brightness control, theme controls, and persistent config via NVS

HARDWARE FACTS (must follow):
- Device: Elecrow CrowPanel Advance 7" 800x480 RGB panel, capacitive touch, ESP32-S3-WROOM-1-N16R8 (16MB flash, 8MB PSRAM).
- Board revision: V1.3.
- V1.3 has a control MCU on I2C address 0x30 controlling backlight brightness, speaker enable, and buzzer.
- V1.3 control MCU commands:
  - Brightness: 0 = max, 244 = min, 245 = off.
  - Buzzer: 246 = on, 247 = off.
- I2C bus uses SDA GPIO15 and SCL GPIO16.
- Known devices on the bus include RTC 0x51, touch 0x5D, and control MCU 0x30.
- Touch controller is GT911-class at I2C 0x5D with INT on GPIO1.
- Use ESP-IDF RGB panel driver and LVGL v8.x.

CURRENT WORKING IMPLEMENTATION NOTES:

Display:
- RGB panel: 800x480.
- Pixel clock: 16 MHz using `LCD_CLK_SRC_PLL240M`.
- Timing: HSYNC 4/40/40, VSYNC 10/30/1, `pclk_active_neg=1`.
- Bounce buffer enabled.
- LVGL vertical compensation: `ver_res = 480 + 40`, `offset_y = -40`.
- Button shadow rendering was reduced to avoid heavy redraw cost during long sessions.

Touch:
- Driver: `esp_lcd_touch` + `esp_lcd_touch_gt911` vendored under `components/`.
- INT wake pulse is required before init on this panel.
- GT911 register address swap should stay disabled for `esp_lcd_panel_io_i2c`.
- Touch long-press timing is configured explicitly in software.
- Keep the current UX conventions intact: tap opens details, long-press opens edit actions, and scroll must never be mistaken for a tap.

Touch calibration strategy:
- Physical height: 480 via `CT_TOUCH_PHYS_V_RES`.
- LVGL target height: 520 via `CT_TOUCH_TARGET_V_RES`.
- Offset and min/max calibration are applied through build flags.

Timezone:
- Chart x-axis labels use local time through the `CT_LOCAL_TZ` build flag.

SOFTWARE STACK:
- ESP-IDF project in PlatformIO.
- LVGL 8.3.x.
- cJSON for JSON parsing.
- `esp_http_client` for HTTPS.
- ESP-IDF certificate bundle for TLS verification.
- NVS for persistence.

IMPORTANT RUNTIME RULES:
- LVGL runs in a dedicated task and UI updates must stay thread-safe.
- All HTTP and JSON parsing must stay off the LVGL task.
- CoinGecko HTTPS requests are serialized to avoid concurrent TLS heap spikes.
- Coin Detail chart loading is async-first with a loading overlay.
- News screen objects are transient and must be cleared on delete events.
- Demo Portfolio toggles should trigger immediate refreshes so real holdings restore cleanly.
